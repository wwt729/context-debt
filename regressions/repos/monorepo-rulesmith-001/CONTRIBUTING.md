# Contributing to rulesmith

Thanks for your interest in contributing to `rulesmith`.

## Before you contribute (required)

All contributions require a signed Contributor License Agreement (CLA) before they can be merged.

This applies to:
- pull requests
- patches
- significant code snippets submitted in issues or discussions (if later merged)

### Why we require a CLA

The CLA helps ensure:
- contributions are legally safe to use
- maintainers can distribute and maintain the project long-term
- the project can be relicensed in the future if needed

## Contribution flow

1. Open an issue (recommended for larger changes).
2. Fork the repository and create a branch.
3. Submit your pull request.
4. Sign the CLA when requested.
5. Maintainer review and merge decision.

## Coding standards

- Keep changes focused and small.
- Add tests when relevant.
- Do not break existing CLI or MCP behavior without documentation.
- Update docs when behavior changes.

## Maintainer authority

`rulesmith` is a maintainer-led project.
Project maintainer(s) have final authority over roadmap, scope, and merge decisions.

## License of contributions

By submitting a contribution, you agree that your contribution is made under the project license (Apache-2.0), subject to the CLA.

See `CLA.md` for the template and terms used by this repository.
